# 🛠 MikroTik: Два внешних IP на разных интерфейсах с одним шлюзом

## 📘 Описание

Задача: настроить MikroTik RouterOS 6.49 таким образом, чтобы:

-   оба выделенных провайдером внешних IP использовались одновременно;
-   каждый прописан на своём интерфейсе;
-   исходящий трафик разных клиентов выходил через разные IP;
-   при этом у провайдера один общий шлюз.

------------------------------------------------------------------------

## 🌐 Исходные данные

  Роль                 Значение
  -------------------- ----------------------------------
  Подсеть провайдера   `91.230.54.128/25`
  Шлюз                 `91.230.54.129`
  IP #1                `91.230.54.144` → `ether1`
  IP #2                `91.230.54.149` → `sfp-sfpplus1`
  Особый клиент        `10.3.0.42` (SEN B24)
  Обычные клиенты      выходят через IP `144`

------------------------------------------------------------------------

## 🧩 Конфигурация MikroTik

### 1️⃣ Добавляем IP-адреса

``` bash
/ip address
add address=91.230.54.144/25 interface=ether1
add address=91.230.54.149/25 interface=sfp-sfpplus1
```

------------------------------------------------------------------------

### 2️⃣ Настраиваем единственный default route

``` bash
/ip route
add dst-address=0.0.0.0/0 gateway=91.230.54.129
```

------------------------------------------------------------------------

### 3️⃣ Маркируем трафик особого клиента

``` bash
/ip firewall mangle
add chain=prerouting src-address=10.3.0.42     action=mark-routing new-routing-mark=to-sfp passthrough=no
```

------------------------------------------------------------------------

### 4️⃣ Отдельный маршрут для SEN B24

``` bash
/ip route
add dst-address=0.0.0.0/0 gateway=91.230.54.129 routing-mark=to-sfp
```

------------------------------------------------------------------------

### 5️⃣ Настраиваем NAT

#### SEN B24 → выходит через IP `91.230.54.149`

``` bash
/ip firewall nat
add chain=srcnat src-address=10.3.0.42 out-interface=sfp-sfpplus1     action=src-nat to-addresses=91.230.54.149
```

#### Остальные клиенты → через `144`

``` bash
add chain=srcnat out-interface=ether1     action=src-nat to-addresses=91.230.54.144
```

------------------------------------------------------------------------

## ✅ Проверка

``` bash
tool traceroute 8.8.8.8 routing-table=to-sfp
```

Обычные клиенты:

``` bash
curl ifconfig.me
# → 91.230.54.144
```

Клиент SEN B24:

``` bash
curl ifconfig.me
# → 91.230.54.149
```

------------------------------------------------------------------------

## 🎯 Итог

Конфигурация позволяет использовать оба IP-адреса независимо, правильно
распределяет исходящий трафик и остаётся совместимой с одним
провайдерским шлюзом.
